# 硬件开发手册：ESP32-C3 核心系统设计

> **设计目标**：确保芯片稳定启动，并兼容多种环境下的程序烧录。

---

## 1. 复位与使能电路 (EN / CHIP_PU)

EN 引脚是 ESP32-C3 的使能端（高电平有效）。为了保证电源稳定后再启动芯片，必须设计 **RC 延迟电路**。

* **核心配置**：
    - **上拉电阻**：使用 **10kΩ** 电阻连接至 3.3V。
    - **滤波电容**：使用 **1uF** 电容连接至 GND。
* **设计逻辑**：利用 RC 充电特性，延迟 EN 引脚达到高电平的时间。如果缺少该电容，在电源缓慢爬升或波动大的环境下，芯片极易死锁（Hang）导致无法开机。

## 2. 下载方案 A：内置 USB 下载 (USB-Serial-JTAG)

ESP32-C3 内部集成了 USB 控制器，通过 **GPIO18 (D-)** 和 **GPIO19 (D+)** 即可实现免串口芯片下载。

### 2.1 USB Type-C 的“5.1k 陷阱” ⚠️
如果你设计的是 USB-C 接口，必须在接口的 **CC1** 和 **CC2** 引脚上分别接一个 **5.1kΩ** 的下拉电阻。

* **为什么要接？** - 这是 USB-C 规范的要求。如果你不接这两个电阻，使用 **C-to-C 线**（如 MacBook 充电线或 PD 充电头）连接你的板子时，电脑将完全检测不到设备。
    - 5.1k 电阻告诉主机端（Host）这是一个受电设备，主机才会开启 VBUS 输出。

## 3. 下载方案 B：外部 UART 下载 (串口芯片)

如果你选择通过外部串口芯片（如 CH340, CP2102）下载，需要注意自动流控电路的设计。

### 3.1 自动下载电路 (Auto-Program)
为了点击“下载”后能自动复位并进入下载模式，通常需要使用两个三极管组成的逻辑电路：
* **输入信号**：DTR 和 RTS（来自串口芯片）。
* **输出控制**：
    - 当 DTR=0, RTS=1 时，EN=1, GPIO9=0（准备进入下载模式）。
    - 当 DTR=1, RTS=0 时，EN=0, GPIO9=1（复位芯片）。

## 4. 启动模式配置 (Strapping Pins)

ESP32-C3 在上电的一瞬间会检测某些引脚的电平，这决定了它是运行 Flash 程序还是进入下载模式。

### 4.1 核心引脚：GPIO9
这是 ESP32-C3 最关键的启动配置引脚。

| 引脚状态 | 启动模式 | 说明 |
| :--- | :--- | :--- |
| **高电平 (Default)** | **SPI Boot** | 从 Flash 启动，正常运行用户程序 |
| **低电平 (Pull-down)** | **Download Boot** | 进入串口/USB下载模式 |

* **设计守则**：
    1. 务必确保 GPIO9 在启动时不会被外设强行拉低。
    2. 建议外部加一个 **10kΩ** 上拉电阻，并设计一个 **BOOT 按键**（另一端接地），用于手动强制进入下载模式。

## 6. 硬件调试 Checklist
- [ ] **USB-C 识别**：CC1/CC2 的 5.1kΩ 电阻是否漏焊？
- [ ] **复位延迟**：EN 引脚是否接了 1uF 左右的电容？
- [ ] **启动电平**：GPIO9 是否在启动瞬间保持高电平？
- [ ] **电源纹波**：ESP32-C3 在 Wi-Fi 开启瞬间电流很大，确保 3.3V 导线足够宽。
